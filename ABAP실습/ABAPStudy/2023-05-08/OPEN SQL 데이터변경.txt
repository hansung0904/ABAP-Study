OPEN SQL에 사용되는 테이블 데이터 변경 명령문에 대해서 알아보자.

1 INSERT 구문
	테이블에 하나 또는 여러 개의 데이터를 삽입. <target>은 테이블 이름으로, 동적으로 선언이 가능하다.
	
	INSERT INTO <target> <lines>.
	<f2> [ASCENDING|DESCENDING] ...
	
	Client를 지정할 수 있으며, 이때는 테이블 이름을 정적/동적으로 선언할 수 있다.
	
		INSERT INTO dbtab [CLIENT SPECIFIED] <lines>.
		INSERT INTO <name> [CLIENT SPECIFIED] <lines>.
		
	1) Single line
		테이블에 하나의 line을 삽입하기 위한 문장이다. <wa>는 테이블과 같은 구조로 선언되어야 한다.
		
		INSERT INTO <target> VALUSE <wa>.
		INSERT <target> FROM <wa>.
		
		INSERT <dbtab> 구문을 사용하려면 TABLES: <dbtab> 선언이 되어야 한다.
			INSERT <dbtab>.
			
	2) Several lines
		인터널 테이블의 모든 값을 한 번에 테이블에 삽입.
		같은 KEY 값을 INSERT 하게 되면, dump error가 발생하게 되는데 이를 방지하기 위해서
		ACCEPTING DUPLICAT KEYS 구문 사용
		INSERT 구문이 실패하면 SY-SUBRC 값 4를 반환
		
		INSERT <target> FROM TABLE <itab>
							[ACCEPTING DUPLICATE KEYS].
							
테이블 데이터 확인하기
	테이블의 데이터를 확인하는 방법은 크게 2가지가 존재
	• T-CODE:SE11(ABAP Dictionary) : 테이블과 같은 오브젝트를 조회/수정/생성하는 용도로 사용
	• T-CODE:SE16(Data Browser) : 데이터를 조회하는 용도로 사용된다.
	
	T-CODE:SE11에서 데이터 조회 버튼을 클릭하면 T-CODE:SE16이 호출
	즉 데이터를 확인할 때에는 그냥 자신에게 익숙한 트랜잭션을 사용하면 된다.
	테이블에 데이터가 추가 되었는지 확인하기 위해 T-CODE:SE16을 실행한다.
	
2. UPDATE 구문
		테이블의 하나 또는 여러 LINE을 변경한다. <target>은 테이블 이름으로, 동적으로 선언이 가능하다.
		
3. DELETE 구문
		테이블에 하나 도는 여러 LINE의 데이터를 삭제한다.
		
4. MODIFY (Interesing or Changing Lines) 구문
			MODIFY는 UPDATE 구문과 INSERT 구문을 합한 기능을 수행한다.
			킷값을 가지는 데이터가 테이블에 존재하면 UPDATE 하고 , 존재하지 않을 때에는 INSERT를 수행한다.
			테이블에 하나 또는 여러 line을 UPDATE 또는 INSERT 할 수 있음
